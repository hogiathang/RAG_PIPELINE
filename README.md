# RAG-based Malware Analysis System

A **Retrieval-Augmented Generation (RAG)** pipeline for automated threat intelligence analysis of JavaScript / Node.js code snippets. The system combines a vector knowledge base (Qdrant), real-time web search (Tavily), and an LLM (Gemini) to produce detailed, structured Threat Intelligence reports.

## Overview

Given a suspicious JavaScript / Node.js code snippet the system:

1. **Retrieves** the most relevant malware examples from a Qdrant vector store (populated from a PDF knowledge base).
2. **Enriches** the context with real-time web search results via Tavily API, guided by targeted queries generated by Gemini.
3. **Generates** a professional Threat Intelligence report (Markdown) using Gemini, covering Threat Summary, Technical Mechanism, External Context, and IoCs.

## Architecture

```
User Input (JS/Node.js code)
        │
        ▼
┌───────────────────┐
│  build_query_     │  → Wraps code into a structured retrieval query
│  prompt()         │
└────────┬──────────┘
         │
         ▼
┌────────────────────────────────────────────┐
│           Retrieval Layer                  │
│  ┌─────────────────┐  ┌──────────────────┐ │
│  │  QdrantAdapter  │  │ WebSearchEngine  │ │
│  │  (vector store) │  │  (Tavily API)    │ │
│  │  BAAI/bge-code  │  │  + Gemini query  │ │
│  │  embeddings     │  │  generation      │ │
│  └────────┬────────┘  └────────┬─────────┘ │
│           └──────────┬─────────┘           │
└──────────────────────┼─────────────────────┘
                       │ contexts: List[str]
                       ▼
┌───────────────────────────────────────────┐
│           Generation Layer                │
│  GeminiAgent (gemini-2.5-flash)           │
│  System prompt: "summary-generation"      │
│  Input: known code examples + web context │
└──────────────┬────────────────────────────┘
               │
               ▼
       Threat Intelligence Report (Markdown)
```

**Key components:**

| Component | Class/Module | Role |
|---|---|---|
| Embedding | `EmbeddingModel` | `BAAI/bge-code-v1` via `sentence-transformers` |
| Vector DB | `QdrantAdapter` | Qdrant (local, `http://localhost:6333`) |
| Web Search | `WebSearchEngine` | Tavily API + Gemini query synthesis |
| LLM | `GeminiAgent` | Gemini 2.5 Flash, multi-token worker pool |
| Retrieval | `retrieve_similar_documents` | Top-5 cosine similarity from Qdrant |
| Generation | `build_prompt_from_retrieve_similar_documents` | Full RAG pipeline |
| Entry Point | `src/main.py` | CLI interface |

All heavy singletons (`EmbeddingModel`, `QdrantAdapter`, `GeminiAgent`, `WebSearchEngine`) are initialised once and reused across calls.

## Configuration

### API Tokens

**`api_tokens/gemini_tokens.txt`** — one Gemini API key per line:

```
AIza...key1
AIza...key2
```

**`api_tokens/search_engine_api_keys.txt`** — one Tavily API key per line:

```
tvly-...key1
tvly-...key2
```

The system automatically rotates keys on rate-limit errors (HTTP 401 / 429 / 432 / 433).

### Embedding Model

Configured in `src/common/embedding_model.py`:

```python
EMBEDDING_MODEL_CONFIG = {
    "model_name": "BAAI/bge-code-v1",   # HuggingFace model ID
    "device": "cpu",                     # Change to "cuda" for GPU
    "normalize_embeddings": True,
    "batch_size": 32,
    "max_length": 512,
}
```

### Qdrant Collection

Configured in `src/common/qdrant_adapter.py`:

```python
QdrantAdapter(url="http://localhost:6333", collection="malware", dim=1536)
```

Change `dim` only if switching embedding models.

## Running the System

### 1. Ingest Documents

Place your PDF knowledge base at `documents/malware-book.pdf`, then run:

```bash
# Foreground (shows progress)
python -m src.ingestion.ingest_data

# Background (recommended for large PDFs)
nohup python -m src.ingestion.ingest_data > ingest.log 2>&1 &
```

This will:
- Load and chunk the PDF (`chunk_size=1000`, `chunk_overlap=200`)
- Generate embeddings via `BAAI/bge-code-v1`
- Upsert into the `malware` Qdrant collection in batches of 100

### 2. Analyze Code Snippets

**From a file:**

```bash
python -m src.main --file path/to/snippet.js

# or via run.sh
./run.sh --file path/to/snippet.js
```

**Inline code:**

```bash
python -m src.main --code "eval(Buffer.from('...', 'base64').toString());"
```

**From stdin (pipe):**

```bash
cat suspicious.js | python -m src.main
```

**Save report to file:**

```bash
python -m src.main --file snippet.js --output reports/report.md
```

**Full usage:**

```
usage: rag-malware-analyzer [-h] [-f PATH | -c CODE] [-o PATH] [--no-banner]

options:
  -h, --help            show this help message and exit
  -f PATH, --file PATH  Path to a JS/Node.js code file to analyze
  -c CODE, --code CODE  Inline code snippet string to analyze
  -o PATH, --output PATH
                        Save the report to a markdown file
  --no-banner           Suppress the startup banner
```

## Pipeline Details

### Retrieval (`src/retrieval/retrieval.py`)

```python
retrieve_similar_documents(question: str) -> list[str]
```

1. Embeds the query using `EmbeddingModel` (`BAAI/bge-code-v1`).
2. Queries Qdrant for **top-5** most similar chunks (cosine similarity).
3. Calls `WebSearchEngine.analyze_code_and_search_web()` which:
   - Sends the code to Gemini with the `code-to-query` prompt to generate 2–3 targeted search queries.
   - Executes each query against the Tavily API.
   - Aggregates URLs, titles, and content snippets.
4. Returns combined list: `qdrant_contexts + web_search_results`.

### Generation (`src/generation/generation.py`)

```python
build_prompt_from_retrieve_similar_documents(user_prompt: str) -> str
```

Formats the retrieved contexts into a structured prompt:

```
[KNOWN CODE EXAMPLES]:
  <user code>

[WEB SEARCH CONTEXT]:
  <context 1>

  <context 2>
  ...
```

Sends it to `GeminiAgent.execute_task(..., "summary-generation")` which uses the **Senior Threat Intelligence Analyst** system prompt.

### GeminiAgent (`src/common/gemini_agent.py`)

- **Singleton** with a thread-safe worker pool (one `Worker` per API key).
- Each worker validates its token on startup by sending a `"ping"` request.
- `execute_task()` acquires the first available worker (60 s timeout) and returns the result.

## Output Format

The Gemini model produces a Markdown report with four sections:

```markdown
## 1. Threat Summary
Executive summary of the malware's purpose and severity.

## 2. Technical Mechanism
Step-by-step breakdown of how the code executes its malicious intent.

## 3. External Context & Known Threats
Correlation with known npm supply-chain attacks or threat actors.

## 4. Indicators of Compromise (IoCs)
List of URLs, IPs, file paths, and strings usable for detection.
```
